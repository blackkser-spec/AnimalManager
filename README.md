# Animal Manager

CLI / GUI / APIの3つのインターフェースを持つ、動物情報管理アプリケーションです。

## 機能
- 動物の登録・削除・一覧表示
- 種類や年齢によるフィルタリング（予定）

## セットアップ
### 1. 依存関係のインストール
1. リポジトリをクローンします。
2. 仮想環境を作成し、有効化します。
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # Windows: .venv\Scripts\activate
   ```
3. 依存関係をインストールします。
   ```bash
   pip install -r requirements.txt
   ```

### CLIモード (コマンドライン)
```bash
python run_cli.py
```
初回起動時に言語設定画面が表示されます。

起動後、メニューから以下の操作が可能です。

- 動物の管理(追加, 削除, 編集等)
- リストの管理(表示, ソート, データクリア)
- 検索機能
- 言語設定の切り替え

### GUIモード (グラフィカルユーザーインターフェース)
```bash
python run_gui.py
```
起動後、以下の設定を選択することが可能です。

  - **使用言語設定**: 対応言語は日本語と英語のみ。
  - **ローカルモード**: ローカルJSONファイルを直接使用します。
  - **リモートモード**: API サーバーを経由して操作、保存ファイルはローカルで管理します。
    - **注意**: リモートモードを使用する場合、事前に別のターミナルで以下のコマンドを実行し、APIサーバー（デフォルト: ポート8080）を起動しておく必要があります。
      ```bash
      python run_api.py
      ```
    - APIを起動すると、ブラウザから `http://127.0.0.1:8080/docs` にアクセスしてAPI仕様を確認・テストできます。

## アーキテクチャ
本アプリは以下のレイヤー構造を採用しています。

- **CLI / GUI / API**: ユーザーインターフェース
- **Controller**: 入力バリデーションと各層の仲介
- **Manager (Service)**: ビジネスロジックの実装
- **Repository**: データの永続化（SQLite / APIクライアント）

```text
[ UI / API ] -> [ Controller ] -> [ Manager ] -> [ Repository ]
```

## 設計方針
本プロジェクトは「責務分離」と「変更容易性」に重きを置きました。
CLI / GUI / API の各UI層は Controller を介して Manager を呼び出す形にし、
ビジネスロジックが UI 実装へ極力依存しないよう設計されています。

また表示メッセージ等は text レイヤーに分離，
キーで管理することにより

- 各UI層での文言の共有
- 多言語対応 (日 / 英) 言語ファイルの追加も可能
- エラーメッセージの統一管理

GUIは起動モードに応じたBackendを使用することで、
ローカルデータとAPI経由データを切り替えて使用可能にしています。
